## 2024-03-10 - IP Spoofing in Rate Limiter\n**Vulnerability:** Rate limiter trusts X-Forwarded-For header blindly, allowing IP spoofing by attackers to bypass rate limits.\n**Learning:** The `request.headers.get('X-Forwarded-For')` header can be easily manipulated by clients if the application is not behind a trusted reverse proxy, or if it doesn't validate that the immediate client is a trusted proxy.\n**Prevention:** Only trust the X-Forwarded-For header if the immediate client IP (`request.client.host`) is a known, trusted proxy.
