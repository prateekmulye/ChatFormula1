## 2025-03-05 - Hash passwords instead of storing plaintext/sha256
**Vulnerability:** API keys were hashed using SHA-256 for storage, which is vulnerable to brute-force attacks compared to slower hashes designed for passwords/secrets.
**Learning:** When migrating to a salted hash like `bcrypt`, `key_hash` can no longer be used for O(1) dictionary lookups because the salt is random and the raw string cannot reliably recreate the lookup key. A split-key format (`prefix_id_secret`) must be used, and the ID part must be generated with `secrets.token_hex` instead of `secrets.token_urlsafe` to prevent the accidental generation of the `_` delimiter. Furthermore, `bcrypt` is CPU-bound, meaning it will block the FastAPI async event loop if run synchronously, so it must be wrapped in `starlette.concurrency.run_in_threadpool`.
**Prevention:** Always use a delimiter-safe encoding (like hex) when constructing multi-part strings, and ensure CPU-bound crypto functions are executed in thread pools within async web frameworks.
