## YYYY-MM-DD - [Eliminate Per-Request StateMachine Compile in LangGraph]
**Learning:** Recompiling a `StateGraph` object (`graph.compile()`) dynamically upon every web request is extremely slow and degrades Time To First Byte (TTFB). LangGraph is designed so that a single `MemorySaver` checkpointer intrinsically manages states and threading natively via `thread_id` values under the hood. There is no need to manually create a dictionary tracking `MemorySaver`s dynamically.
**Action:** Always pre-compile the state machine natively at application spin-up into a global configuration `app_state['agent_graph'].compiled_graph` and use `ainvoke({"configurable": {"thread_id": "YOUR_THREAD"}})` to route session data explicitly to the global `MemorySaver` without recreating logic or incurring per-request compilation initialization costs.
